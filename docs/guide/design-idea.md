# 设计理念

## 极致的类型安全

Annil 的初衷是提供极致的类型安全体验,在此基础上衍生出新的组件构建方式和组件类型系统。

## 新的组件构建方式

Annil 在构建组件（页面）时采用“根组件 + 子组件”的组合方式。这对ts类型最友好,也使组件更易管理。

- 根组件(RootComponent)可以理解为wxml中的最外层元素或一个抽象的概念。
  负责声明外部传值(properties)、组件事件(customEvents)、维护组件公共状态、公共方法等。
  this(类型)上不会有子组件状态和方法。

- 子组件是当前组件使用到的自定义组件或指定的wxml元素。
  通过CustomComponent/ChunkComponent 为子组件与根组件的交互提供支持。
  this(类型)上可以获取到根组件的状态和方法。

## 组件类型系统

以往组件库会用文档来描述组件属性、事件等信息，开发者在使用时只能靠记忆或查文档来获取这些信息，且无法得到类型提示和类型检查。Annil 组件API设计时,就支持导入和生成组件类型,在使用第三方库时,也能得到类型提示和类型检查,极大提升开发体验和代码质量。

## 类型补全

微信小程序官方对ts类型迭代较慢，Annil 提供了一些遗漏的类型,或把某些API做了重新包装(语法糖),以提供更好的类型体验。
